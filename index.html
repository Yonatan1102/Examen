Análisis de Requisitos

Guía de estudio sobre UML, Casos de Uso, Historias de Usuario y Diagramas de Actividades

Tema Principal

El tema central del contenido es la utilización de UML para el análisis, modelado y especificación de requisitos de software. Se estudian los diagramas de casos de uso, historias de usuario y diagramas de actividades para representar correctamente las necesidades del sistema.

Temas Estudiados

UML

Lenguaje gráfico estandarizado utilizado para modelar sistemas de software.

Casos de Uso

Representan funcionalidades del sistema desde la perspectiva de los usuarios.

Historias de Usuario

Describen necesidades utilizando la estructura: Como..., Quiero..., Para...

Diagramas de Actividades

Permiten modelar procesos, flujos de trabajo y algoritmos.

Definiciones Importantes

Análisis de Requisitos

Proceso para identificar y comprender las necesidades del sistema.

Especificación de Requisitos

Documento que describe detalladamente funcionalidades y restricciones.

Actor

Persona o sistema externo que interactúa con el sistema.

Caso de Uso

Funcionalidad ofrecida por el sistema a un actor.

Include

Permite reutilizar comportamientos comunes entre casos de uso.

Extend

Agrega comportamiento opcional a otro caso de uso.

Historia de Usuario

Descripción breve de una necesidad desde la perspectiva del usuario.

Criterios de Aceptación

Condiciones necesarias para considerar terminada una funcionalidad.

Diagrama de Actividades

Representación gráfica de procesos y flujos de trabajo.

Requerimiento Funcional

Acción o funcionalidad que el sistema debe realizar.

Requerimiento No Funcional

Características de calidad como seguridad, rendimiento o usabilidad.

Generalización

Relación de herencia entre actores o casos de uso.
